# Line 컴포넌트 사용 가이드 예시

이 문서는 `Dynamic Chart 1.0` 플러그인에서 `Line Chart`를 사용하는 방법을 설명하기 위한 예시 문서입니다.  
기준 컴포넌트는 Figma의 [`Type=line`](https://www.figma.com/design/PNdGxZ4OupoY1zF1KwWH7q/Chart--%EC%9E%91%EC%97%85%EC%9A%A9-?node-id=3177-23661&m=dev) 노드입니다.

## 1. Line 컴포넌트란

Line 컴포넌트는 시간의 흐름, 구간별 변화량, 여러 시리즈의 추세를 비교할 때 사용하는 차트입니다.

- 각 행은 하나의 라인 시리즈입니다.
- 각 열은 데이터 포인트입니다.
- `Line BG`를 사용하면 라인 아래 면적을 함께 표현할 수 있습니다.
- `guide line`을 사용하면 최소값, 최대값, 평균값, 중앙값 기준선을 함께 표시할 수 있습니다.

## 2. 사용 전 체크

플러그인이 정상 동작하려면 차트의 기본 구조가 유지되어야 합니다.

- 차트 루트는 `Type=line` 차트 컴포넌트 또는 인스턴스여야 합니다.
- 차트 내부에 `chart`, `chart_main`, `y-axis`, `chart_legend` 구조가 있어야 합니다.
- 컬럼은 `col-01`, `col-02` 같은 이름 규칙을 따라야 합니다.
- 각 컬럼 내부에는 `tab`, `x-empty`가 있어야 합니다.
- 각 라인 시리즈 컨테이너는 `line-01`, `line-02`처럼 2자리 번호 규칙을 따라야 합니다.
- 각 시리즈 컨테이너 내부에는 `line`, `fill`, `fill_top`, `fill_bot` 레이어가 있어야 합니다.
- 보조선은 `Asst_line_min`, `Asst_line_max`, `Asst_line_avg`, `Asst_line_ctr` 이름 규칙을 따라야 합니다.

권장 방식은 디자인 시스템의 원본 `Line` 컴포넌트를 복제해서 사용하는 것입니다. 필수 레이어 이름을 임의로 바꾸면 플러그인이 라인을 찾지 못할 수 있습니다.

## 3. 기본 사용 절차

### 3.1 차트 선택

1. Figma에서 `Line` 차트 전체 컴포넌트 또는 인스턴스를 선택합니다.
2. 플러그인을 실행합니다.
3. `Select Chart Type`에서 `Line Chart`를 선택합니다.

기존 차트에 다시 적용할 때도 가능하면 차트 전체를 선택하는 것을 권장합니다.

### 3.2 Graph Setting 설정

`Line Chart`를 선택하면 아래 항목을 설정할 수 있습니다.

- `Graph Col`
  - 라인이 지나가는 구간 수입니다.
  - Line 차트는 실제 데이터 열 수가 `Graph Col + 1`로 계산됩니다.
  - 예를 들어 `Graph Col = 4`이면 데이터 포인트는 5개가 필요합니다.
- `Cell Count`
  - Y축을 몇 개 구간으로 나눌지 결정합니다.
- `Mark Count`
  - 라인 시리즈 개수입니다.
  - 값이 3이면 3개의 라인이 그려집니다.
- `Y Min`, `Y Max`
  - Y축 범위를 제어합니다.
- `Thickness`
  - 라인 두께입니다.

### 3.3 Data Setting 입력

데이터 그리드에 각 시리즈의 값을 입력합니다.

- 한 행은 하나의 라인입니다.
- 한 열은 하나의 데이터 포인트입니다.
- 행 헤더는 범례 텍스트로 사용됩니다.
- 열 헤더는 X축 라벨로 사용됩니다.

예시:

| 항목 | 1월 | 2월 | 3월 | 4월 | 5월 |
| --- | ---: | ---: | ---: | ---: | ---: |
| 가입자 수 | 12 | 18 | 15 | 24 | 29 |
| 전환율 | 8 | 11 | 10 | 13 | 17 |

위 예시는 라인 2개, 데이터 포인트 5개이므로:

- `Mark Count = 2`
- `Graph Col = 4`

로 설정하면 맞습니다.

## 4. 데이터 모드 이해하기

플러그인은 `Raw`와 `%` 두 가지 모드를 지원합니다.

### 4.1 Raw 모드

- 실제 수치 데이터를 입력할 때 사용합니다.
- Y축 최소값은 현재 구현상 `0`으로 고정됩니다.
- `Y Max`를 비워두면 데이터 최대값을 기준으로 자동 계산됩니다.
- `Y Max`를 직접 입력하면 자동값보다 작은 경우에도 실제 적용값은 데이터 최대값 이상으로 보정됩니다.

### 4.2 % 모드

- 비율, 점유율, 전환율처럼 0~100 범위 데이터를 표시할 때 사용합니다.
- 현재 구현상 Y축 범위는 항상 `0~100`으로 동작합니다.
- 따라서 `%` 모드에서는 `Y Min`, `Y Max` 입력값을 사용자 정의 범위로 쓰지 않습니다.

## 5. Style 설정

Line 차트는 `Mark`와 `Plot Area` 영역을 나누어 스타일을 조정합니다.

### 5.1 Mark

각 라인 시리즈의 스타일입니다.

- `Stroke`
  - 라인 색상, 선 종류, 두께를 설정합니다.
- `Fill`
  - 라인 위 포인트가 있을 경우 포인트 색상에 사용됩니다.
- `Line BG`
  - 라인 아래 면적 색상을 설정합니다.
  - `Visible`을 끄면 면적 채우기를 숨길 수 있습니다.

### 5.2 Plot Area

차트 배경과 보조 영역 스타일입니다.

- `Background`
  - 셀 배경 스타일입니다.
- `X-axis line`
  - 셀 상단 라인 스타일입니다.
- `Border`
  - 플롯 영역의 외곽선 스타일입니다.
- `guide line`
  - 보조선의 표시 여부와 시각 스타일을 설정합니다.

## 6. guide line 사용법

UI에는 `guide line`으로 표시되지만, 내부 구현 명칭은 `assist line`입니다.

사용 방법:

1. Preview 헤더 또는 Style 패널에서 `guide line`을 `ON`으로 켭니다.
2. 필요한 항목만 선택합니다.
3. Style 탭에서 색상, 선 종류, 두께를 조정합니다.

선택 가능한 항목:

- `min`: 전체 데이터의 최소값
- `max`: 전체 데이터의 최대값
- `avg`: 전체 데이터의 평균값
- `ctr`: 현재 Y축 범위의 중앙값

보조선 텍스트는 컴포넌트 내부의 `Asst_line_data` 프로퍼티를 통해 갱신됩니다. 보조선 컴포넌트를 새로 만들 때 이 속성이 빠지면 숫자 텍스트가 자동으로 바뀌지 않을 수 있습니다.

## 7. Figma 구조 규칙

플러그인은 시각적으로 비슷한 레이어를 찾는 방식이 아니라, 이름 규칙을 기준으로 구조를 해석합니다.

필수 구조 예시:

| 레이어 이름 | 역할 |
| --- | --- |
| `y-axis` | Y축 라벨 영역 |
| `y_cel-01` ~ `y_cel-n` | Y축 셀 및 라벨 |
| `col-01` ~ `col-n` | 각 구간 컬럼 |
| `tab` | 컬럼 본체 |
| `x-empty` | X축 라벨 표시 영역 |
| `line-01` ~ `line-n` | 시리즈별 라인 컨테이너 |
| `line` | 실제 선 인스턴스 |
| `fill` | 면적 채우기 인스턴스 |
| `fill_top` | 선 위쪽 영역 |
| `fill_bot` | 선 아래쪽 영역 |
| `chart_legend` | 범례 영역 |
| `Asst_line_min` 등 | 보조선 영역 |
| `chart_container` | 보조선/플롯 영역 정렬 기준 |

특히 Line 차트는 각 컬럼 안에서 시리즈 컨테이너를 `line-01`, `line-02`처럼 정확한 이름으로 찾아서 데이터를 주입합니다.  
이 이름이 깨지면 `Line apply failed: required line structure is missing.` 오류가 발생할 수 있습니다.

## 8. 자주 발생하는 문제

### 8.1 라인이 그려지지 않는 경우

가능한 원인:

- `line-01` 같은 시리즈 컨테이너가 없음
- 컨테이너 안에 `line` 또는 `fill`이 없음
- `fill_top`, `fill_bot` 레이어가 없음
- 선 인스턴스에 `direction` variant가 없음

### 8.2 X축 라벨 수가 맞지 않는 경우

Line 차트는 `Graph Col + 1`개의 데이터 포인트를 사용합니다.  
따라서 `Graph Col` 값과 실제 데이터 열 수가 맞지 않으면 의도한 형태와 다르게 보일 수 있습니다.

### 8.3 guide line 값이 이상한 경우

다음 항목을 먼저 확인합니다.

- 현재 모드가 `Raw`인지 `%`인지
- `Y Max`가 자동인지 수동인지
- 입력 데이터에 빈칸이나 의도하지 않은 `0`이 있는지

## 9. 권장 작업 방식

- 원본 마스터를 직접 수정하기보다 인스턴스 복제본에 적용하는 것을 권장합니다.
- 구조 수정이 필요하면 먼저 원본 `Line` 컴포넌트의 레이어 이름을 유지한 상태에서 작업합니다.
- 새 Line 변형을 만들 때는 기존 `Type=line`을 복제한 뒤 필요한 시각 표현만 바꾸는 것이 안전합니다.
- 스타일만 바꾸고 싶다면 `Style Tab`에서 먼저 확인하고, 구조 변경은 최소화하는 것이 좋습니다.

## 10. 문서에 함께 넣으면 좋은 안내 문구

아래 문구는 실제 사용자용 문서 첫머리에 바로 넣어도 됩니다.

> Line 차트는 추세 비교용 컴포넌트입니다. `Graph Col`은 구간 수를 의미하며, 실제 데이터 열 수는 항상 `Graph Col + 1`입니다. 구조 레이어 이름(`col-01`, `line-01`, `fill_top`, `Asst_line_max` 등)을 변경하면 플러그인이 데이터를 주입하지 못할 수 있으니 원본 구조를 유지해주세요.
